Proton Library Manager Updates

20th October 2008

 

Library Manager:     Version 2.2.1.2

 

Fixed error in positioning Library Manager on multi-screen set ups.  Minor changes to support the Pre-Compiler.

 

Pre-Compiler            Version 1.3.3.10

 

The Pre-Compiler can now be set to generate statistics relating to the use of variables in your code and to report back via the IDE on any variables which have been declared but never used in the code.  This is a useful feature if you are running close to the limit of your RAM.  Both features are selectable via the Pre-Compiler set up dialog and are disabled by default.

 

To view the variable statistics report you will also need to install the Statistics Viewer plug-in.  This will present formatted list of variable usage and identify which banks the variables are placed.

 

 

02nd March 2008

 

NOTE- The Library Manager Installation has now been divided into 2 separate installs.

 

Library Manager comprises:

Library Manager and Pre-Compiler

Library Developer comprises:

Macro Generator, Macro Extractor, Library Help File Builder and Library Packer.

 

Library Manager:     Version 2.2.1.1       

 

Minor changes have been made to the Library Manager in support of the Pre-Compiler changes.  The Help file for Library Manager has been updated.

 

Library Manager will now position itself relative to the IDE.  Thus if you reposition the IDE, when the Library Manager is next invoked it will appear on screen in the same position relative to the IDE.  If Library Manager is running when the IDE is moved it will not follow the position and you will have to reposition it manually.

 

Fixed a problem with Library Add function not working.

 

Pre-Compiler            Version 1.3.3.5

 

Pre-Compiler has been modified to support the latest version of the PDS compiler.  It is intended to work with version 3.3.0.6 or later of the Compiler.  (It will still operate with earlier versions of the compiler and will detect the version of the compiler installed and act accordingly).

 

An additional optional function has been added to the Pre-Compiler which will delete any error files generated by a previous compile of the PIC application.  It will only delete error files which are associated with the current compile.  This is useful as old error messages appear as results of a current compile which can lead to confusion.  This option can be turned on or off from the Pre-Compiler Setup dialog.  Its default state is Off – i.e. files will not be deleted, however it is recommended that this option is turned on.

 

Some minor changes have been made to the user interface of the Pre-Compiler, the most significant being that it will now list all library commands that have been added to the current compile.

 

The Pre-Compiler’s code parser has been rewritten to remove a bug which could under some situations wrongly detect a command in the source text.  As a by-product the Pre-Compiler now runs a little faster.

 

Macro Generator:    Version 1.2.5.0

 

Change to the error message wording for return value from macro.  (Numbers cannot be returned from a subroutine, only variables.)

 

 

16th March 2007

 

Library Manager:     Version 2.1.2.1        Pre-Compiler            Version 1.3.3.3

 

Minor internal changes have been made to Library Manager and a correction to the Precompiler to handle specific instances of variable and label name clashes.  Revised version of Library Packer is also included which resolves an error where the library file was being installed into the wrong library directory.

 

Macro Generator:    Version 1.2.4.2

 

Changes to correct the macro code generated when the return parameter is optional.

Now takes the Macro name as the base name for the Help TopicID i.e. “MACRONAME.HTM”.

 

Macro Extractor:     Version 1.2.1.0

 

Changes have been made to improve the detection of macros under all conditions including macros which do not reference a subroutine and macros which use the MACRO-/ENDM- notation.

 

There are improved instructions in some steps of the extractor wizard.

 

If an existing library file is chosen you will be asked if you want to overwrite it or append to the existing file or you can cancel out from this dialog.

 

After identifying all required macros you will have the choice of letting Macro Extractor save the file then open it in the IDE or pasting the file into a new page in the IDE without first saving it.  If you choose the latter you will have to save it manually in your selected library folder.

 

A new applet Library HelpFile Builder has been added which will build a Project File for use with the Microsoft™ HTML Help Workshop application.  Using this applet in conjunction with the MSWord® Help File Template provided will make the job of producing Library Help files a little easier and more predictable.

 

29th June 2006

 

Library Manager:     Version 2.0.1.0

 

This is a major update to the User Interface of Library manager.  There is no change in functionality from the previous version.

 

The new interface provides an Explorer like interface to the Library Manager.  The libraries are now show as a tree, double clicking on a node will expand the node to show all the macros within the selected library.

 

The manager can be resized and positioned as required and these values will be held next time the library manager is run.

 

The help files have also been updated.

 

28th May 2006

 

Pre-Compiler:           Version 1.3.2.2

 

Fixes an error when referencing a library file from within your source.

 

19th May 2006

 

Pre-Compiler:           Version 1.3.1.2

 

The Pre-Compiler now supports the definition of variables after they have been used in the code.  Know as Forward Declarations, if you prefix a variable with Rem Fwd you will be able to successfully compile this program even if you have used that variable in your code before declaring it.

 

This is useful when a library file uses a specific I/O pin as it avoids the need to add the generated library as an Include file in your code after declaring the pin.  This previous method using the include file is still supported.

 

This feature can be enabled or disabled from the Library Manager set up dialog.

 

 

11th April 2006

 

Help Template:

 

Minor changes to correct formatting and added an Operators subsection.

 

HTML Help Workshop Support:

 

A skeleton Help Project file has been added which can be opened using Microsoft’s™ HTML  Help Workshop.  The file is called Lib_Help_Project_Template.hhp and is included in the Library Tools Plug-in folder.  If you have the HTML workshop you simply have to add in the HTML help files you have created and compile and you will have produced a compiled HTML Help file.

 

Macro Generator:    Version 1.2.3.1

 

Fixes:

 

If you are including a help file with your macro the browse for Help file now only saves the File Name and not the path.  Library manage assumes you will be saving the help file in the save folder as the library file.

 

If you define a macro with no parameters the Macro Generator will now include the code to jump to the subroutine defined in the macro.

 

An ampersand has been added to the special macro calls when using passing label references and loading data into the FSR.  This is necessary as the Proton compiler is not specifically aware of these macros.

 

Library Packer:        Version: 1.1.2.0

 

Fixes:

The message box asking you to confirm the location to save the packed library file now shows the correct path.

 

The library packer set up file is now saved to the same location as the packed file.

 

Delete now deletes the correct file.

 

16th March 2006

 

Pre-Compiler:           Version 1.3.1.0

 

New:

The Pre-Compiler Icon, when minimised, will now change to reflect the On or Off state of the Pre-Compiler.

 

A new option has been added in Set Up to disable the Pre-Compiler from checking if a library file has been updated.  This option is only required for developers and should not be invoked in normal usage.  On restart it will always default to Off.

 

Fixes:

Pre-Compiler properly identifies macro’s which are either disabled or do not apply to the current target.

 

When loading or reloading the library file, the Pre-Compiler will now check the format of the loaded and file.  If the number of fields in the records differ from the expect count an error will be reported and the file will not be reloaded.

 

Library Manager:     Version 1.3.1.0

 

New:

Added a “What’s New” under the Help menu.  This will invoke Explorer.  From now on this will provide a running history of changes to all applications in the Library Manager suite.

 

Fixes:

When the Library Manager loads the library file, it will now check the field count in the library records is the expected count.  If the count differs from the expected count the Library Manager will rebuild the library and save it.  It will report Library Built in the status bar.

 

Library Manager will correctly identify the location of the Library File when started outside the IDE.  If the IDE has not been installed Library Manager will report that the IDE is missing and will close down.

 

Macro Generator:    Version 1.2.1.0

 

Fixes:

When generating the code to pass Chars, Strings and Arrays to the macro, a “#ENDIF” statement is omitted causing the macro to fail when compiled.  This has now been fixed.

 

When passing Chars, Strings and Arrays to the macro, the “.mcd” Hint text generated did not identify the type for these variables.  This has now been resolved.

14th March 2006

 

Previous Release

 

What’s New:

This release is primarily a maintenance release, however, the following additional features have been added as a result of user’s feedback.

The Pre-Compiler Set up dialog can now also be reached from the Library Manager.  The existing Pre-Compiler Setup will still be available on the Plug-In menu.

The Pre-Compiler operational state is now reported in the status bar of Library Manager.  It is also reported in the Title of the Pre-Compiler when it is minimised and on the form when it is restored.

A Main Menu has been added to Library Manager.  This reflects the functions on the Toolbar with the addition of a standard Windows File, Exit.

Help files have been updated to reflect the above changes.

Fixes:

List out of bounds message has been fixed when right clicking on an empty macro entry in Library Manager.  This has necessitated the removal of the title row above the macro list.

The very first compile of an application which uses a Library file referenced within the application fails.  This has now been fixed.

Library Packer/Unpacker will not now update an existing library file unless the user confirms its OK to update.  If the user declines to update a specific library file the Library Manager will not automatically rebuild the library list.
